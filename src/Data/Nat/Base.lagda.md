# Natural numbers {defines="natural-number"}

The natural numbers are the inductive type generated by `zero`{.Agda}
and closed under taking `suc`{.Agda}cessors. Thus, they satisfy the
following induction principle, which is familiar:

```agda
Nat-elim : ‚ąÄ {‚Ąď} (P : Nat ‚Üí Type ‚Ąď)
         ‚Üí P 0
         ‚Üí ({n : Nat} ‚Üí P n ‚Üí P (suc n))
         ‚Üí (n : Nat) ‚Üí P n
Nat-elim P pz ps zero    = pz
Nat-elim P pz ps (suc n) = ps (Nat-elim P pz ps n)

iter : ‚ąÄ {‚Ąď} {A : Type ‚Ąď} ‚Üí Nat ‚Üí (A ‚Üí A) ‚Üí A ‚Üí A
iter zero f = id
iter (suc n) f = f ‚ąė iter n f
```

Translating from type theoretic notation to mathematical English, the
type of `Nat-elim`{.Agda} says that if a predicate `P` holds of
`zero`{.Agda}, and the truth of `P(suc n)` follows from `P(n)`, then `P`
is true for every natural number.

## Discreteness

An interesting property of the natural numbers, type-theoretically, is
that they are `discrete`{.Agda}: given any pair of natural numbers,
there is an algorithm that can tell you whether or not they are equal.
First, observe that we can `distinguish`{.Agda} zero from successor:

```agda
zero‚Č†suc : {n : Nat} ‚Üí ¬¨ zero ‚Č° suc n
zero‚Č†suc path = subst distinguish path tt where
  distinguish : Nat ‚Üí Type
  distinguish zero = ‚ä§
  distinguish (suc x) = ‚ä•
```

<!--
```agda
suc‚Č†zero : {n : Nat} ‚Üí ¬¨ suc n ‚Č° zero
suc‚Č†zero = zero‚Č†suc ‚ąė sym
```
-->

The idea behind this proof is that we can write a predicate which is
`true`{.Agda ident=‚ä§} for zero, and `false`{.Agda ident=‚ä•} for any
successor. Since we know that `‚ä§`{.Agda} is inhabited (by `tt`{.Agda}),
we can transport that along the claimed path to get an inhabitant of
`‚ä•`{.Agda}, i.e., a contradiction.

```agda
pred : Nat ‚Üí Nat
pred 0 = 0
pred (suc n) = n

suc-inj : {x y : Nat} ‚Üí suc x ‚Č° suc y ‚Üí x ‚Č° y
suc-inj = ap pred
```

Furthermore, observe that the `successor`{.Agda} operation is injective,
i.e., we can ‚Äúcancel‚ÄĚ it on paths. Putting these together, we get a
proof that equality for the natural numbers is decidable:

## Arithmetic

::: warning
**Heads up!** The arithmetic properties of operations on the natural
numbers are in the module [`Data.Nat.Properties`].
:::

[`Data.Nat.Properties`]: Data.Nat.Properties.html

Agda already comes with definitions for addition and multiplication of
natural numbers. They are reproduced below, using different names, for
the sake of completeness:

```agda
plus : Nat ‚Üí Nat ‚Üí Nat
plus zero y = y
plus (suc x) y = suc (plus x y)

times : Nat ‚Üí Nat ‚Üí Nat
times zero y = zero
times (suc x) y = y + times x y
```

These match up with the built-in definitions of `_+_`{.Agda} and
`_*_`{.Agda}:

```agda
plus‚Č°+ : plus ‚Č° _+_
plus‚Č°+ i zero y = y
plus‚Č°+ i (suc x) y = suc (plus‚Č°+ i x y)

times‚Č°* : times ‚Č° _*_
times‚Č°* i zero y = zero
times‚Č°* i (suc x) y = y + (times‚Č°* i x y)
```

The exponentiation operator `^`{.Agda} is defined by recursion on the
exponent.

```agda
_^_ : Nat ‚Üí Nat ‚Üí Nat
x ^ zero = 1
x ^ suc y = x * (x ^ y)

infixr 10 _^_
```

## Ordering

We define the order relation `_‚Č§_`{.Agda} on the natural numbers
by appealing to the decision procedure `_‚Č§?_`{.Agda}.

```agda
record _‚Č§_ (x y : Nat) : Type where
  constructor lift
  field
    lower : So (x ‚Č§? y)
```

We could also define the relation by recursion on the numbers to be
compared or as an inductive predicate. However, our definition has
the benefit of being a *definitional* [[proposition]].

```agda
‚Č§-is-prop : {x y : Nat} ‚Üí is-prop (x ‚Č§ y)
‚Č§-is-prop p q = refl
```

As a further optimization, `_‚Č§?_`{.Agda} is implemented using the `BUILTIN` decision
procedure `_<?_`{.Agda}. This makes it possible to typecheck proofs of `_‚Č§_`{.Agda}
nearly instantly, even when the numbers involved are quite large.

We define the strict ordering on `Nat`{.Agda} as well,
re-using the definition of `_‚Č§_`{.Agda}.

```agda
_<_ : Nat ‚Üí Nat ‚Üí Type
m < n = suc m ‚Č§ n
infix 7 _<_ _‚Č§_
```

As an "ordering combinator", we can define the _maximum_ of two natural
numbers by recursion: The maximum of zero and a successor (on either
side) is the successor, and the maximum of successors is the successor of
their maximum.

```agda
max : Nat ‚Üí Nat ‚Üí Nat
max zero zero = zero
max zero (suc y) = suc y
max (suc x) zero = suc x
max (suc x) (suc y) = suc (max x y)
```

Similarly, we can define the minimum of two numbers:

```agda
min : Nat ‚Üí Nat ‚Üí Nat
min zero zero = zero
min zero (suc y) = zero
min (suc x) zero = zero
min (suc x) (suc y) = suc (min x y)
```
